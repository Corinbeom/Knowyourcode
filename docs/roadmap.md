# KnowYourCode Roadmap

## 최근 피드백 요약

- 분석 정확도는 보통 수준이지만 질문과 피드백은 코드 이해에 도움이 된다는 반응이 있었다.
- 질문이 엔티티/모델 계층에 쏠리면 서비스 흐름을 확인하기 어렵다.
- 질문에 답할 때 관련 코드를 미리 볼 수 있으면 답변 부담이 줄어든다.
- 실무 코드 리뷰에서는 변경 의도뿐 아니라 책임 분리, 예외 처리, 테스트, 회귀 위험을 함께 묻는다.

## 제품 방향

- Repo Mode: 내 프로젝트 구조와 요청/데이터 흐름을 설명할 수 있는지 검증한다.
- Commit Mode: 내가 작성한 변경사항을 코드 리뷰에서 방어할 수 있는지 검증한다.

## 작업 우선순위

1. 질문 다양성 개선
   - 요청 흐름, 서비스 로직, 데이터 흐름, 변경 영향도, 면접/리뷰 관점이 섞이도록 한다.
   - 같은 계층 파일만 반복해서 질문이 생성되지 않게 한다.

2. 질문별 코드 미리보기
   - 퀴즈 화면에서 현재 질문의 관련 파일 excerpt를 함께 보여준다.
   - Commit Mode에서는 diff patch를 먼저 보여줘 코드 리뷰 상황에 가깝게 만든다.

3. Commit Mode 리뷰형 강화
   - 변경 의도, 영향 범위, 테스트/리스크, 리뷰형 질문을 명확히 분리한다.
   - 책임 분리, 예외 처리, 기존 구조와의 일관성, 회귀 가능성을 질문에 반영한다.

4. 운영 안정화
   - 실패한 분석/평가는 quota를 차감하지 않는다.
   - API 서버는 Nginx allowlist로 필요한 엔드포인트만 FastAPI로 전달한다.
   - 공식 도메인을 `https://knowyourcode.cloud`로 고정한다.

## 완료 기준

- 실제 저장소 2개 이상에서 질문 5개가 엔티티/모델에만 쏠리지 않는다.
- Repo/Commit Quiz 화면에서 질문별 관련 코드가 표시된다.
- Commit Mode 질문에 테스트/리스크와 리뷰형 관점이 포함된다.
- 정상 분석 성공 시에만 Redis quota가 증가한다.
